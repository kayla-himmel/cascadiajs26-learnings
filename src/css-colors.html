CSS Color Functions

Modern CSS ships powerful built-in color tools — no preprocessor required. These demos show four functions covered at CascadiaJS 2026, with live examples you can tweak directly in the source files.

light-dark()

Returns one of two color values depending on the active color-scheme — light or dark. Define your palette once; the browser picks the right value automatically. No media queries, no duplicate rules.

Live demo

This whole page is themed with light-dark(). Toggle the theme in the top nav to watch it switch.

Code snippet

:root {
  color-scheme: light dark;
  /* Try changing these color values and saving! */
  --surface-primary: light-dark(#f5f5f5, #1a1a1a);
  --text-primary:    light-dark(#111111, #f0f0f0);
}

/* Force a specific theme regardless of OS preference */
[data-theme="light"] { color-scheme: light; }
[data-theme="dark"]  { color-scheme: dark; }

color-mix()

Mixes two colors in a given color space. Perfect for generating hover states, tints, and shades on the fly — without a preprocessor or JavaScript.

Code snippet

.demo {
  /* Try changing --demo-color to any color and saving! */
  --demo-color: #0066cc;
}

.swatch { background: color-mix(in srgb, var(--demo-color) 75%, white); }
/* Change the percentage to control how much of each color shows */

contrast-color()

Automatically picks black or white — whichever has better contrast — against a given background color. Eliminates manual contrast math and ensures readable text on any background.

Experimental — not yet widely supported. See the caniuse link below.

Live demo

Note: contrast-color() is experimental. If text is not showing above, your browser may not support it yet. Check caniuse for support status.

Code snippet

.button {
  /* Try changing this color — the text will always stay readable */
  background: var(--btn-color);
  color: contrast-color(var(--btn-color));
}

Modern CSS Color Notation

Modern CSS drops the commas between color channels and uses a / separator before the alpha value. This is CSS Color Level 4 syntax — cleaner, more consistent, and now widely supported.

Live demo

Before — comma syntax

rgba(225, 65, 65, 0.9)
hsla(215, 80%, 60%, 0.9)

After — space-separated syntax

rgb(225 65 65 / 0.9)
hsl(215 80% 60% / 0.9)

The swatches are identical — same rendered color, different syntax. Spot the difference: commas between channels vs. spaces, and / before the alpha value.

Code snippet

/* Before — old syntax uses rgba()/hsla() for colors with alpha */
.old { color: rgba(225, 65, 65, 0.9); }
.old { color: hsla(215, 80%, 60%, 0.9); }

/* After — new space-separated syntax (same result) */
.new { color: rgb(225 65 65 / 0.9); }
.new { color: hsl(215 80% 60% / 0.9); }
